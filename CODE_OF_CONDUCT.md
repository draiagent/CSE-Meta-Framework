# Code of Conduct

## Purpose

CSE Meta Framework is an open technical project. Participation must remain professional, respectful, and focused on improving the framework.

## Expected Behavior

Contributors must:

- Discuss ideas, evidence, and implementation rather than personal attributes
- Provide clear and constructive review comments
- Acknowledge uncertainty and limitations
- Respect licensing, privacy, and security requirements
- Avoid harassment, threats, discrimination, or targeted hostility
- Avoid publishing confidential or restricted information
- Follow maintainer decisions after the documented review process is complete

## Unacceptable Behavior

The following are not acceptable:

- Personal attacks
- Harassment or intimidation
- Deliberate disruption
- Misrepresentation of validation results
- Secret or sensitive-data disclosure
- Repeated scope expansion without governance review
- Bypassing security or release controls

## Enforcement

Maintainers may edit, reject, lock, or remove contributions that violate this policy. Repeated or severe violations may result in temporary or permanent exclusion from project spaces.

## Reporting

Report conduct concerns privately to the project maintainers. Do not publish personal or sensitive details in public Issues.

## Scope

This Code of Conduct applies to Repository discussions, Issues, Pull Requests, reviews, releases, and other official CSE Meta Framework project spaces.
